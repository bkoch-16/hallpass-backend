Hallpass API Demo
About this demo ▴

This is a portfolio project demonstrating a backend REST API. There is no full frontend, this tool lets you explore the API endpoints directly, inspect responses, and interact with the data layer.

Getting started: Go to Auth > Sign In and use one of the seed accounts below. The session cookie is then used automatically by all protected endpoints.

Seed accounts (password: password): student@hallpass.dev   teacher@hallpass.dev   admin@hallpass.dev   superadmin@hallpass.dev

Creating a pass: Passes can only be created during an active school period, so a few things must be in place first:

  • The school calendar needs an entry for today with a schedule type. As an admin, use Schools-API > Calendar > Upsert Calendar Entries with today's date. The current time (school timezone) must fall within one of that schedule type's periods, otherwise the API returns "No active period".
  • The request body needs a valid destinationId (see Schools-API > Destinations > List Destinations) and accepts an optional note.
  • A student creates a pass for themselves and it starts as PENDING, waiting for teacher approval. A teacher or admin creates a pass on behalf of a student (studentId required) and it is automatically approved: it goes straight to ACTIVE, or WAITING if the destination is at capacity.
  • A student can only have one in-flight pass at a time, and the school's pass policy may cap how many passes a student gets per day.
  • Prod credentials are not normally shared publicly, but are provided here to offer a more stable demo environment.
  • Cold starts may cause the first request to take a few seconds. In a real production setting I would set minimum instances to 1 or more to prevent this startup time, but I keep them at 0 in a portfolio to keep infrastructure costs minimal.
