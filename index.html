Hallpass API Demo
About this demo ▴

What this is: Hallpass is a digital hall-pass system: students request passes to leave class, teachers approve them, and admins manage schedules, destinations, and policies. It's a backend portfolio project — there is no full frontend, so this tool lets you call the API endpoints directly and inspect their responses.

Getting started: Go to Auth > Sign In and use one of the seed accounts below; the session cookie is then sent automatically with all protected requests. Each account demonstrates a different permission level. (Sharing live credentials like this is unusual, but deliberate here to keep the demo stable.)

Seed accounts (password: password): student@hallpass.dev   teacher@hallpass.dev   admin@hallpass.dev   superadmin@hallpass.dev

Try it — create a pass: Passes can only be created during an active school period, so set that up first:

  1. As an admin, use Schools-API > Calendar > Upsert Calendar Entries to add an entry for today, choosing a schedule type whose periods cover the current time (in the school's timezone). Without this, creating a pass returns "No active period".
  2. Grab a destinationId from Schools-API > Destinations > List Destinations.
  3. Create the pass with that destinationId (and an optional note).

Pass rules: A pass created by a student starts as PENDING, awaiting teacher approval; one created by a teacher or admin on a student's behalf (studentId required) is automatically approved and goes straight to ACTIVE — or WAITING if the destination is at capacity. A student can only have one in-flight pass at a time, and the school's pass policy may cap passes per day.

Note: The first request may take a few seconds due to a cold start. In real production I would set minimum instances to 1 or more; here they stay at 0 to keep infrastructure costs minimal.
